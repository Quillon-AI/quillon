ПРОЦЕСС

От отклика до оффера

В среднем 7–10 дней от первого контакта до оффера. Без бессмысленных тестовых, без 5 этапов согласования.

  1. 01

    Знакомство

    Присылаешь CV или короткое сообщение в Telegram. Отвечаем в течение дня — назначаем тех-собес или говорим, что не совпадаем сейчас.

  2. 02

    Тех-собеседование

    1 час с senior-инженером команды. Не «расскажи об ООП», а разбор реальных задач из нашего бэклога. Live-coding опционально.

  3. 03

    Тестовое

    Только если нужно — даём небольшую задачу из реального продукта. 4–8 часов работы, оплачивается фикс если кандидат не прошёл дальше.

  4. 04

    Финальное

    Встреча с CTO или CEO. Обсуждаем оффер, проект, ожидания, культуру. Без HR-вопросов о «5 годах через 5 лет».

  5. 05

    Оффер

    За день после финального собеса. Договариваемся о дате выхода, готовим железо, делаем onboarding-план под первый месяц.

ПОЧЕМУ QUILLON

Что получаешь
в команде

01 / Production

Реальные продукты с первого дня

Никаких «учебных задач». Уже на второй неделе — коммиты в production-репозитории, ревью своего кода, ответственность за конкретный модуль.

02 / Code review

Культура ревью по существу

Не формальный approve. Разбор архитектуры, производительности, читаемости. Учишься у senior’ов с опытом высоконагруза.

03 / Стек

Свой стек, без legacy

Python 3.11+, FastAPI, Django 5, Flutter, RAG, vector DB. Никаких 10-летних PHP-монолитов и «поправь в jQuery, потом переедем».

04 / Гибрид

Офис Казань + удалёнка

2–3 дня в офисе для тех, кому важно живое общение. Полная удалёнка для mid/senior-инженеров на стабильных продуктах.

05 / Open road-map

Прозрачные цели

Road-map обсуждается открыто. Любой инженер может предложить feature или редизайн архитектуры — обсуждение по существу, не по иерархии.

06 / AI-native

AI как инструмент команды

Code-review с AI, автогенерация тестов, документация через LLM, RAG для поиска по базе знаний. Ты не конкурируешь с AI — ты его используешь.
